The object manager uses mappings to determine what the default implementation is for a class in a constructor (scope dependent). For this goal where designed preference node in the di.xml settings file. For example: In class A here are dependencies in the constructor - interface B and class C. Our aim to get in the class A concrete realization of interface B - BConcrete and replace concrete implementation for class C on its child CSpecific. We can easily determine for object manager to instantiate interface B as concrete class BConcrete by using this template. In the same way, we can determine for object manager to instantiate class CSpecific instead of class C.
Predefined variables will take the following values:
${FOR}   Target PHP Class to change its concrete implementation
${TYPE}   PHP Class that will be instantiated instead of specified in predefined ${FOR} variable
